# THEME: FEAT-fetcher-progress-feedback

## 类型
FEAT (Feature)

## 背景
全文抓取功能存在以下问题：
1. 失败率高，用户无法了解具体失败原因
2. 页面反馈差，点击"抓取全文"后只显示"抓取中..."，无进度信息
3. 失败时前端没有显示错误信息
4. 批量抓取进度不透明，状态仅存内存

## 目标
1. 优化全文抓取的用户体验
2. 提供清晰的进度展示
3. 显示详细的错误反馈
4. 实现状态可视化

## Non-goals
- 不改变 trafilatura 核心抓取逻辑
- 不引入新的抓取库
- 不修改数据库 schema（除非必要）

## 当前阶段
Research

Active
